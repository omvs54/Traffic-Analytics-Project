Traffic Data Model Deliverables

1. Star Schema Design

The data model follows a Star Schema architecture, optimized for analytical queries and reporting. It consists of a central Fact table containing metrics, surrounded by Dimension tables containing context.

Fact Table: Fact_Traffic

Contains the quantitative metrics (Vehicle counts) and foreign keys linking to dimensions.

Column NameTypeDescription
TrafficID (PK)INTUnique identifier for the traffic record (derived from original ID).
DateKey (FK)INTForeign key linking to Dim_Date.
LocationKey (FK)INTForeign key linking to Dim_Location.
VehicleCountINTThe measured number of vehicles (Fact).

Dimension Table: Dim_Date

Enables time-based analysis and supports the time hierarchy.

Column NameTypeDescription
DateKey (PK)INTSurrogate key (e.g., YYYYMMDDHH).
FullDateTimeDATETIMEOriginal timestamp.
YearINTYear component.
MonthVARCHARMonth name/number.
DayINTDay of the month.
HourINTHour of the day (0-23).
DayOfWeekVARCHARName of the day (e.g., Monday).

Dimension Table: Dim_Location

Contains location details and supports the location hierarchy (Region → Street → Intersection).

Column NameTypeDescription
LocationKey (PK)INTSurrogate key for the location.
JunctionIDINTOriginal Junction ID (1-4).
RegionVARCHARBroader location grouping (e.g., North, South).
StreetVARCHARStreet name (e.g., "Main St").
IntersectionVARCHARSpecific intersection point.

2. Relationships & Data Integrity

Relationships are established using Foreign Keys (FK) in the Fact table referencing Primary Keys (PK) in the Dimension tables to ensure referential integrity.

  • Fact_Traffic (DateKey) → Dim_Date (DateKey): Many-to-One relationship. Ensures every traffic record corresponds to a valid time slot.
  • Fact_Traffic (LocationKey) → Dim_Location (LocationKey): Many-to-One relationship. Ensures every traffic record corresponds to a valid location configuration.

3. Hierarchies

Hierarchies allow for drilling down into data during analysis (e.g., from Year down to Hour).

  • Time Hierarchy: Year → Month → Day → Hour
  • Location Hierarchy: Region → Street → Intersection
