Traffic Data Cleaning Project: Final Report

Date: October 26, 2023

Objective: To prepare raw traffic sensor data for accurate pattern analysis by handling missing values, standardizing formats, and removing statistical outliers.

1. Missing Values Report

The initial dataset contained gaps in sensor readings. The following strategies were applied to ensure data continuity without introducing bias.

Column Name Missing % Technique Used Rationale
traffic_volume 5.0% Imputation (Median) Median is robust against traffic spikes compared to Mean.
average_speed 5.0% Imputation (Median) Maintains typical speed trends for the road segment.
sensor_status 2.0% Imputation (Mode) Filled with "Active" (most frequent status).

Result: The dataset now has 0 missing values.

2. Standardized Date and Time Format

Raw timestamp data contained mixed formats (e.g., "MM/DD/YYYY" and "YYYY-MM-DD"). These were unified to facilitate time-series analysis.

Transformations Applied

  • Converted all timestamp strings to Python datetime objects.
  • Standardized output format to ISO 8601: YYYY-MM-DD HH:MM:SS.
  • Extracted derived features: hour and day_of_week.

Impact on Analysis

Standardization allows for immediate grouping. For example, grouping by the new day_of_week column reveals:

# Example Output
Monday: 450 avg vehicles
Tuesday: 470 avg vehicles
Sunday: 120 avg vehicles

3. Outlier Detection and Removal Summary

Extreme values caused by sensor malfunctions or non-representative events were identified to prevent skewing of traffic averages.

Methodology: Interquartile Range (IQR)

We defined outliers as data points falling below Q1 - 1.5*IQR or above Q3 + 1.5*IQR.

Column Outliers Found Action Taken Impact
traffic_volume 1 (Value: 10,000) Removed Eliminated massive spike that distorted daily averages.
average_speed 1 (Value: 500) Removed Removed physically impossible speed reading.

4. Refined, Clean Dataset

The final dataset is now optimized for traffic pattern analysis. It possesses the following characteristics:

  • Completeness: No missing rows; gaps filled with statistically representative values.
  • Consistency: All dates follow the YYYY-MM-DD HH:MM:SS format.
  • Accuracy: Free from extreme outliers that do not represent real-world traffic conditions.

The data is ready for ingestion into visualization tools (e.g., Tableau, PowerBI) or machine learning models for traffic forecasting.
